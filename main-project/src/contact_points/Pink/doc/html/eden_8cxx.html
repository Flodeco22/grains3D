growth and/or shrinking of an 2D binary image - Eden process

Usage: eden in.pgm niter grow shrink topo out.pgm

Description:

Algorithm is as follows.

For growth:

  • start from known set
  • take pixels from the border into a queue
  • choose randomly one such pixel
  • set it to 1
  • update queues
  • repeat

For shrinking:

  • same thing but set to 0

For adding boundary noise

  • alternate growth and shrinking

With topological constraints

  • make sure the point to add or remove is simple

Parameters:

  • niter (positive integer): number of iterations
  • grow (1/0): perform growing or not
  • shrink (1/0): perform shrinking or not
  • topo (26/6/8/4/0): connectivity for the object - 0: no topological constraint

Types supported: byte 2d
