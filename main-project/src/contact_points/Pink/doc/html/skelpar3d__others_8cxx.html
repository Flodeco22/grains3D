parallel 3D binary skeleton

Usage: skelpar3d in.pgm algorithm nsteps [inhibit] out.pgm

Description: Parallel 3D binary thinning or skeleton. The parameter nsteps gives, if positive, the number of parallel thinning steps to be processed. If the value given for nsteps equals -1, the thinning is continued until stability.

The parameter algorithm is a numerical code indicating which method will be used for the thinning. The possible choices are:

  • 0: Palagyi-Kuba (curvilinear, 6 directions, 1998)
  • 1: Palagyi (curvilinear, sequential, 2006)
  • 2: Palagyi (surface, parallel directional, 2002)
  • 3: Palagyi (surface, fully parallel, 2008)
  • 4: Raynal (curvilinear, directional, 2011)
  • 5: Raynal (surface, directional, 2011)
  • 6: Lohou-Bertrand (curvilinear, symmetric, 2007)
  • 7: Ma-Wan-Chang (curvilinear, 2 subfields, 2002)
  • 8: Tsao-Fu (curvilinear, 6 directions, 1982)
  • 9: Ma-Sonka (curvilinear, fully parallel, does not preserve topology 1996)
  • 10: Ma-Wan (curvilinear (18/6) 6 directions, CVIU 2000)
  • 11: Lohou-Bertrand (curvilinear 6 directions, DAM 2005)
  • 12: Lohou-Bertrand (curvilinear 12 directions, DAM 2004)
  • 13: ACK3a with isthmuses- see lskel_ACK3a.c
  • 14: CKSC - see lskeletons.c
